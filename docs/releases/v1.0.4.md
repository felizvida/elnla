# BenchVault v1.0.4

This prerelease focuses on code-audit hardening for backup evidence, local file
confinement, and clearer failure reporting.

## Changes

- Constrained all backup metadata paths used by the viewer, integrity checker,
  thumbnails, readable/search sidecars, and attachment restore to configured
  backup roots.
- Rejected absolute paths, drive-letter paths, UNC-style paths, `..` traversal,
  sibling-prefix tricks, and symlink targets that resolve outside the selected
  backup folder.
- Hardened attachment restore fallback logic so attachment names are treated as
  metadata, not trusted filesystem paths.
- Verified the local integrity ledger as a chained log. Removing, editing, or
  reordering an earlier ledger entry now prevents later entries from being
  treated as locally corroborated.
- Replaced the all-failed backup message with category-aware guidance. True
  owner-rights failures still explain the NIH/NICHD PI-owner rule, while
  network, authorization, storage, extraction, verification, and setup failures
  now report their own cause.
- Added `docs/trust_safety_and_correctness.md`, a plain-language trust model for
  backup correctness, tamper-evidence, path confinement, read-only LabArchives
  access, and operational safeguards.
- Updated the quickstart, limitations, documentation index, and README release
  references for the hardening release.

## Verification

- `flutter test`
- `flutter test test/backup_service_settings_test.dart`
- `flutter analyze`
- `scripts/release_smoke_check.sh`
- `git diff --check`
- secret, credential, local absolute-path, and tracked-local-data scans

The release workflow builds the unsigned macOS prerelease zip and publishes
`SHA256SUMS.txt` for the GitHub Actions-built artifact.

## Important Limits

- BenchVault provides tamper-evidence, not true immutable storage or legal
  certification.
- Local integrity evidence should be anchored externally for high-stakes records
  review.
- The macOS app artifact remains unsigned and not notarized.
- Windows and iPad remain scaffolded but unvalidated as release platforms.
