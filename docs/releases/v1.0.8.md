# BenchVault v1.0.8

This prerelease sharpens BenchVault's first-run trust language and makes the
main desktop experience feel more like a protected notebook viewer than a
backup console.

## Changes

- Replaced the stacked status/preflight/search banners with a single Notebook
  Protection health strip.
- Renamed the primary backup action to `Back Up Eligible Notebooks` to reflect
  NIH/NICHD owner-only backup rules.
- Added a blocking `Local copy not verified` reader state for changed, missing,
  or unexpected protected files, with a deliberate `Open Unverified Copy`
  override.
- Reframed attachment export language around `Save Original` instead of
  restore/write-back terminology.
- Made local search and OpenAI search states clearer, with local fallback
  language and a refreshed AI search screenshot.
- Improved compact/narrow-window behavior by capping the search panel and
  making the Protected Notebooks pane scroll as one surface.
- Refreshed the quickstart source and PDF, trust/correctness notes, limitations,
  platform checklist, README, and developer reference to match the product
  language.

## Release Assets

The tag workflow uploads:

- `BenchVault-macOS-v<version>-prerelease.zip`
- `SHA256SUMS-macOS.txt`
- `BenchVault-Windows-v<version>-prerelease.zip`
- `SHA256SUMS-Windows.txt`
- `BenchVault-iPadOS-v<version>-unsigned-validation.zip`
- `SHA256SUMS-iPadOS-validation.txt`

The iPadOS artifact is an unsigned validation build for Apple signing review. It
is not installable on iPad without Apple Developer signing, provisioning, and an
approved distribution path such as TestFlight or managed app deployment.

## Important Limits

- macOS remains the only platform exercised as the primary local target.
- Windows has CI build/package validation, but still needs real workstation
  testing, installer packaging, and signing.
- iPadOS has CI build/package validation, but still needs Apple signing,
  entitlement review, Files app workflow testing, and real-device validation.
