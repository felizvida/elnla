# BenchVault v1.0.3

This prerelease borrows the safest useful ideas from the local reference
projects without changing BenchVault's local-first, read-only design.

## Changes

- Added attachment original-version and thumbnail metadata to rendered backups.
- Added safe thumbnail display in the read-only viewer when LabArchives backups
  include thumbnail payloads.
- Added SQLite backup-layout parsing through the local `sqlite3` command when
  JSON export tables are absent.
- Added per-notebook queue position and timing metadata to backup run outcomes.
- Added an exclusive local backup lock so manual and scheduled backups cannot
  overlap.
- Added experimental macOS LaunchAgent install/uninstall scripts for closed-app
  scheduled backup testing.
- Added search-hit part highlighting when opening a result in the viewer.
- Documented lessons from the local reference projects and updated limitations.

## Verification

- `flutter test`
- `flutter test test/backup_parser_test.dart`
- `flutter analyze`
- `scripts/release_smoke_check.sh`
- `bash -n scripts/install_macos_launch_agent.sh`
- `bash -n scripts/uninstall_macos_launch_agent.sh`
- secret and absolute-path scans across tracked public files

The release includes `SHA256SUMS.txt` for the GitHub Actions-built macOS zip.

## Important Limits

- The macOS app artifact remains unsigned and not notarized.
- SQLite parsing depends on a local `sqlite3` command and compatible
  LabArchives backup schema.
- The LaunchAgent scripts are experimental developer/support tooling, not a
  signed packaged background helper.
- Windows and iPad remain scaffolded but unvalidated as release platforms.
