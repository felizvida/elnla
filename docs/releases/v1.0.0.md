# BenchVault v1.0.0

This is the first BenchVault GitHub prerelease. It is a macOS-first development
preview for LabArchives GOV backup, integrity checking, audit export, offline
viewing, and local/OpenAI-assisted notebook search.

## Highlights

- Read-only LabArchives GOV backup flow with explicit production endpoint
  allowlist.
- Backup Center preflight, run manifests, classified skip reasons, and notebook
  status cards.
- Offline read-only viewer with breadcrumbs, page outline, attachment evidence,
  integrity warnings, and attachment restore.
- Local fuzzy search with optional OpenAI answer mode and filters for text,
  files, comments, exact phrase, and verified backups.
- SHA-256 integrity sealing plus local audit export as Markdown, JSON, CSV, and
  external hash-anchor text.
- Printable quickstart PDF and full implementation limitations documentation.

## Verification

- `flutter test`
- `flutter analyze`
- `scripts/release_smoke_check.sh`
- `flutter build macos`
- secret and absolute-path scans for public files and the generated PDF

## Important Limits

- The macOS app artifact is not signed or notarized yet.
- Credentials are still stored in local ignored files, not macOS Keychain.
- Windows and iPad are scaffolded but not validated as supported release
  platforms.
- BenchVault provides tamper-evidence, not legal certification or true immutable
  storage by itself.
