# BenchVault v1.0.7

This prerelease stabilizes the new cross-platform CI/CD work by making
backup-relative metadata paths portable across macOS, Windows, and iPadOS
validation builds.

## Changes

- Stores backup-relative metadata paths with forward slashes in generated
  records, render files, readable exports, search indexes, and integrity data.
- Resolves those portable paths back to the native platform separator when the
  desktop app opens, verifies, or restores backed-up attachments.
- Keeps Windows build validation and prerelease zip packaging in GitHub
  Actions.
- Keeps iPadOS no-codesign build validation and unsigned validation packaging
  in GitHub Actions.
- Keeps automatic GitHub prerelease creation on `v*` tags before asset upload.

## Release Assets

The tag workflow uploads:

- `BenchVault-macOS-v<version>-prerelease.zip`
- `SHA256SUMS-macOS.txt`
- `BenchVault-Windows-v<version>-prerelease.zip`
- `SHA256SUMS-Windows.txt`
- `BenchVault-iPadOS-v<version>-unsigned-validation.zip`
- `SHA256SUMS-iPadOS-validation.txt`

The iPadOS artifact is an unsigned validation build for Apple signing review. It
is not installable on iPad without Apple Developer signing, provisioning, and an
approved distribution path such as TestFlight or managed app deployment.

## Verification

- `flutter test`
- `flutter analyze`
- `scripts/release_smoke_check.sh`
- `scripts/package_macos_release.sh 1.0.7`
- GitHub Actions CI on macOS, Windows, and iPadOS build validation
- GitHub Actions release packaging for macOS, Windows, and iPadOS validation
- secret, credential, local absolute-path, and tracked-local-data scans

## Important Limits

- macOS remains the only platform exercised as the primary local target.
- Windows has CI build/package validation, but still needs real workstation
  testing, installer packaging, and signing.
- iPadOS has CI build/package validation, but still needs Apple signing,
  entitlement review, Files app workflow testing, and real-device validation.
