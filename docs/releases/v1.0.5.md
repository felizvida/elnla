# BenchVault v1.0.5

This prerelease expands BenchVault's GitHub CI/CD from a macOS-only path into a
cross-platform validation and packaging pipeline.

## Changes

- Added GitHub Actions Windows build validation on `windows-latest`.
- Added GitHub Actions iPadOS no-codesign build validation on macOS runners.
- Updated the tag release workflow so it creates or updates the GitHub
  prerelease automatically before uploading assets.
- Added Windows prerelease packaging with
  `scripts/package_windows_release.ps1`.
- Added unsigned iPadOS validation packaging with
  `scripts/package_ipados_validation.sh`.
- Updated macOS packaging to publish `SHA256SUMS-macOS.txt`, allowing each
  platform artifact to have its own checksum file without release-asset
  collisions.
- Updated the README, limitations, and platform release checklist to describe
  Windows and iPadOS release status accurately.

## Release Assets

The tag workflow uploads:

- `BenchVault-macOS-v<version>-prerelease.zip`
- `SHA256SUMS-macOS.txt`
- `BenchVault-Windows-v<version>-prerelease.zip`
- `SHA256SUMS-Windows.txt`
- `BenchVault-iPadOS-v<version>-unsigned-validation.zip`
- `SHA256SUMS-iPadOS-validation.txt`

The iPadOS artifact is an unsigned validation build for Apple signing review. It
is not installable on iPad without Apple Developer signing, provisioning, and an
approved distribution path such as TestFlight or managed app deployment.

## Verification

- `flutter test`
- `flutter analyze`
- `scripts/release_smoke_check.sh`
- `scripts/package_macos_release.sh 1.0.5`
- `bash -n scripts/package_ipados_validation.sh`
- YAML syntax inspection through GitHub Actions workflow execution
- secret, credential, local absolute-path, and tracked-local-data scans

## Important Limits

- macOS remains the only platform exercised as the primary local target.
- Windows has CI build/package validation, but still needs real workstation
  testing, installer packaging, and signing.
- iPadOS has CI build/package validation, but still needs Apple signing,
  entitlement review, Files app workflow testing, and real-device validation.
